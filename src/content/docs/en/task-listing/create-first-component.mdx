## Angular CLI

You've already used the CLI to run our application locally with the `ng serve` command in the <a href="/legacy-course/getting-started/#running-the-application" target="_blank">Getting Started</a> section.

Now use it to create a new component as the Angular CLI also provide a dedicated command to generate a new component, `ng generate component`.

##### Instructions

<Steps>

1. Run the following command in a terminal to generate a new component called `task-list`

   ```bash
   ng generate component task-list
   ```

   or the short version:

   ```bash
   ng g c task-list
   ```

2. Look for the generated folder called `task-list` in the `src/app` directory.

    ```diff
    src
    └─ app
    +   └─ task-list
    +       └─ task-list.ts
    +       └─ task-list.html
    +       └─ task-list.css
    ```

</Steps>

## Component Metadata

When opening the `task-list.ts` file, you can see the following code:

```typescript
// task-list.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-task-list',
  imports: [],
  templateUrl: './task-list.html',
  styleUrls: ['./task-list.css']
})
export class TaskList {

}
```

---

The **@Component** decorator defines the component's configuration metadata.

<NotificationCard state="info" title="Note">
  A decorator is a way to do meta-programming. They can modify their target
  (classes, methods, ...), call other methods when the target is called, or add
  metadata intended for a framework. This last point corresponds to how we use
  them in Angular.
</NotificationCard>

---

The **selector** property defines the component's HTML tag name.
By using this selector in an HTML Template (view), Angular creates and displays an instance of the component.

```typescript ins="selector: 'app-task-list',"
// task-list.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-task-list',
  imports: [],
  templateUrl: './task-list.html',
  styleUrls: ['./task-list.css']
})
export class TaskList {

}
```

---

The **templateUrl** property defines the location of the HTML Template (its view) for the component.
This is the view that will be displayed when the CSS **selector** is used in another HTML Template.

```typescript ins="templateUrl: './task-list.html',"
// task-list.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-task-list',
  imports: [],
  templateUrl: './task-list.html',
  styleUrls: ['./task-list.css']
})
export class TaskList {

}
```

---

The **styleUrls** property defines the locations of the component's CSS files.
These styles will be applied to the component's template.

```typescript ins="styleUrls: ['./task-list.css']"
// task-list.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-task-list',
  imports: [],
  templateUrl: './task-list.html',
  styleUrls: ['./task-list.css']
})
export class TaskList {

}
```

<NotificationCard title="Global styles" state="info" >
  Global styles can be defined in the `src/styles.css` file.
</NotificationCard>

## Display the Component

So far, the application only displays the HTML content of the **App** component in the browser.
You've generated the new component but it's still unused as Angular does no know where to use it: you need to use its selector `app-task-list` to display its HTML template.

As its selector is not a valid native HTML tag like `<button>`, we need to explain Angular its origin, otherwise it will display an error message:

```
✘ [ERROR] NG8001: 'app-task-list' is not a known element:
1. If 'app-task-list' is an Angular component, then verify that it is included in the '@Component.imports' of this component.
2. If 'app-task-list' is a Web Component then add 'CUSTOM_ELEMENTS_SCHEMA' to the '@Component.schemas' of this component to suppress this message. [plugin angular-compiler]

    src/app/app.html:5:0:
      5 │ <app-task-list/>
        ╵ ~~~~~~~~~~~~~~~~

  Error occurs in the template of component App.

    src/app/app.ts:7:15:
      7 │   templateUrl: './app.html',
        ╵                ~~~~~~~~~~~~
```

To do this, you need to import the component in the `app.ts` file to get its reference:

```diff lang="ts"
// app.ts
+ import { TaskList } from './task-list/task-list';
```

Then add it to the `imports` array of the **@Component** decorator for the component to understand it's part of its context:

```ts ins="TaskList"
// app.ts
@Component({
    // ...
    imports: [TaskList],
    })
      export class App {
    }
```

##### Instructions

<Steps>

1. Update the `template`property of the `app.ts` file as follows:

    ```ts ins={"import the TaskList component": 5-6} ins={"Add the TaskList component to the App component imports": 10-11}
    // app.ts
      import { Component } from '@angular/core';
      import { RouterOutlet } from '@angular/router';


      import { TaskList } from './task-list/task-list';

      @Component({
      selector: 'app-root',

      imports: [RouterOutlet, TaskList],
      templateUrl: './app.html',
      stylesUrl: ['./app.css'],
    })
      export class App {
      protected title = 'angular-introduction-course';
    }
    ```

2. Update the `app.html` file to display the `task-list` component:

    ```html ins={"Add the component selector, and the <main> tag for proper HTML semantics and CSS management": 4-7}
     <div class="navbar bg-base-100 shadow-sm">
       <a class="btn btn-ghost text-xl">Task Manager</a>
     </div>

     <main class="m-12 max-w-screen-md mx-auto">
       <app-task-list />
     </main>
    ```

3. Check the changes in your browser.

</Steps>
